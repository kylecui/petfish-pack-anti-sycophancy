# Council Output Schema

Council 的整体输出必须遵循此 schema。适用于完整模式。

## Markdown Schema

```markdown
## Council 判断

### 1. 问题重述

真正要判断的是：……

### 2. 五个顾问的判断

#### 反对者

核心判断：……
关键理由：
- ……
对决策的影响：……
我不知道：……

#### 本质思考者
……

#### 机会挖掘者
……

#### 局外人
……

#### 执行者
……

### 3. 交叉审查

重复或低价值观点：
- ……

真正有价值的观点：
- ……

### 4. 去掉弱观点

删除：
- ……

保留：
- ……

### 5. 综合结论

我的判断是：……

最大风险是：……

最大机会是：……

下一步应该：……

我不知道的是：……
```

## Field Descriptions

| Field | Required | Description |
|---|---|---|
| `问题重述` | Yes | 用户真正要判断的问题，不机械复述原话。 |
| `五个顾问的判断` | Yes | 包含 Critic、Essence、Opportunity、Outsider、Executor 五段独立判断。 |
| `交叉审查` | Yes | 识别重复、低价值、高价值观点。 |
| `去掉弱观点` | Yes | 明确列出删除了什么、保留了什么。 |
| `综合结论` | Yes | Arbiter 的最终判断，包含判断、最大风险、最大机会、下一步、不知道。 |

## Constraints

- 五个顾问的判断不能重复。
- 综合结论不能是五个观点的平均值。
- 必须出现"我不知道"的部分。
- 下一步必须包含至少一个可立即执行的动作。
- 弱观点必须被明确删除，不能保留在最终结论中。

## Quick Mode Variant

当使用快速模式时，输出可压缩为：

```markdown
## Council 快速判断

反对者：……
本质思考者：……
机会挖掘者：……
局外人：……
执行者：……

删掉弱观点后，结论是：……

下一步：……

我不知道：……
```

快速模式仍需满足：观点不重复、至少一个真实风险、至少一个可立即执行动作、明确"我不知道"。
