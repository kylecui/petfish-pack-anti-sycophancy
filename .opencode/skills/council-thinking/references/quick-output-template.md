# Council 快速判断 — 快速模式输出模板

## Council 快速判断

反对者：……

本质思考者：……

机会挖掘者：……

局外人：……

执行者：……

删掉弱观点后，结论是：……

下一步：……

我不知道：……

---

### 使用说明

快速模式适用于：

- 用户要求简洁回答。
- 时间有限或信息有限。
- 问题相对单一，不需要完整五步拆解。

即使使用快速模式，也必须满足：

- 五个顾问不能重复说同一件事。
- 至少指出一个真实风险。
- 至少给出一个可立即执行的动作。
- 明确说出"我不知道"的部分。

---

## 快速 vs 完整模式决策树

```
用户请求
  │
  ├─ 用户说"快速""简要""一句话"？ ──→ 快速模式
  │
  ├─ 问题只有1个决策维度？ ──→ 快速模式
  │
  ├─ 输入信息 < 3句话？ ──→ 降级模式（只输出缺什么+一个验证动作）
  │
  └─ 其他情况 ──→ 完整模式
```

## 快速模式的边界

快速模式牺牲的是**篇幅**，不是**质量**。以下不能省略：

| 不能省略 | 原因 |
|----------|------|
| 问题重述（1句） | 不重述就跑题 |
| 反对者判断 | 没有反对者=迎合 |
| 删弱观点后的结论 | 没有筛选=堆砌 |
| 下一步动作 | 没有行动=空谈 |
| "我不知道" | 没有边界=乱猜 |

可以省略的：交叉审查过程（直接给删弱结果）、机会/局外人/执行者的详细理由（一句话带过）。

## 快速模式失败信号

如果快速模式输出中出现以下任一信号，升级为完整模式：

- 反对者找不到真实漏洞 → 信息不足，需要完整拆解
- 五个顾问有3个以上方向一致 → 可能是表面共识，需要交叉审查
- 结论比输入还模糊 → 信息不够，切降级模式
